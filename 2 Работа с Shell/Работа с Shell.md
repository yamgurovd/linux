## Используемые команды из инструкций раздела 2 (весь блок)

| Команда                                         | Результат / Описание                                                                              | Пояснение                                                                                                                              |
|-------------------------------------------------|---------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------|
| `sudo apt update`                               | Обновляет список пакетов из репозиториев                                                        | Подготовка к установке и обновлению программного обеспечения                                                                           |
| `sudo apt install -y git`                       | Устанавливает Git без подтверждения                                                             | Установка системы контроля версий Git                                                                                                  |
| `git clone https://github.com/rotoro-cloud/linux-vm.git` | Клонирует удалённый репозиторий в текущий каталог                                               | Получение материалов для практики                                                                                                      |
| `sudo bash linux-vm/start.sh`                   | Запускает скрипт подготовки виртуальной машины с root-правами                                  | Автоматическая настройка среды                                                                                                         |
| `pwd`                                           | Показывает полный путь текущей рабочей директории                                               | Помогает определить, где пользователь находится в файловой системе                                                                    |
| `ls`                                            | Отображает список файлов и каталогов в текущей директории                                       | Просмотр содержимого директории                                                                                                        |
| `ls -l`                                         | Показывает подробный (long) список с правами, владельцем и датами                               | Используется для получения детальной информации о файлах и каталогах                                                                  |
| `ls -a`                                         | Показывает все файлы, включая скрытые (начинающиеся с точки)                                    | Полезно для просмотра скрытых системных файлов                                                                                         |
| `ls -ld`                                        | Показывает информацию именно о текущей директории, а не её содержимом                           | Используется для изучения атрибутов самой директории                                                                                   |
| `ls -ld /home/d`                                | Показывает подробную информацию о каталоге `/home/d`                                           | Позволяет проверить права и владельца конкретного каталога                                                                             |
| `mkdir Toyota Audi BMW Ford`                     | Создаёт каталоги с указанными именами                                                          | Быстрый способ создания нескольких каталогов                                                                                            |
| `mkdir -p Camry/2018`                            | Создаёт вложенную структуру каталогов; все недостающие каталоги создаются автоматически          | Удобно для быстрой организации дерева каталогов                                                                                        |
| `cd Toyota`                                      | Переход в каталог `Toyota`                                                                      | Изменение текущего рабочего каталога                                                                                                   |
| `cd temp`                                        | Переход в каталог `temp`                                                                        |                                                                                                                                        |
| `cd ..`                                          | Переход на один уровень вверх в иерархии каталогов                                             | Позволяет подниматься по дереву каталогов                                                                                             |
| `cd`                                             | Переход в домашний каталог пользователя                                                        | Быстрый возврат в домашнюю директорию                                                                                                 |
| `mv X1/ X5`                                      | Переименование или перемещение каталога `X1` в `X5`                                           | Команда mv отвечает как за перемещение, так и за переименование; слеш перед именем каталога может влиять на результат                  |
| `rm -r <dir>`                                    | Рекурсивное удаление каталога и его содержимого                                               | Аккуратно, удаляет каталог со всем содержимым                                                                                          |
| `cat filename`                                   | Вывод содержимого файла на экран                                                               | Используется для быстрого просмотра содержимого файла                                                                                   |
| `touch file.txt`                                 | Создание пустого файла или обновление времени последнего доступа                               | Позволяет быстро создать новый файл                                                                                                   |
| `echo "text" > file.txt`                         | Запись текста в файл (перезаписывает содержимое)                                               | Перенаправление вывода в файл                                                                                                         |
| `more filename`                                  | Постраничный просмотр содержимого файла                                                        | Удобно для чтения больших текстов                                                                                                     |
| `less filename`                                  | Расширенный постраничный просмотр с прокруткой                                                | Лучше, чем `more` для навигации по текстам                                                                                            |
| `find ~/Toyota -name test1.txt`                   | Поиск файла `test1.txt` в каталоге `~/Toyota` и его подкаталогах                              | Помогает быстро найти нужные файлы по имени                                                                                            |
| `lsblk`                                          | Выводит список блоковых устройств (дисков, разделов)                                          | Помогает увидеть структуру подключенных накопителей                                                                                   |
| `lscpu`                                          | Показывает информацию о процессоре                                                           | Включает архитектуру, количество ядер, модель и другую информацию                                                                     |
| `lsmem`                                          | Показ информации о блоках памяти                                                             | Детализированная информация о конфигурациях памяти                                                                                   |
| `lshw`                                           | Детальное описание оборудования                                                             | Полный аппаратный профиль системы                                                                                                    |
| `history`                                        | История введённых команд                                                                     | Позволяет просмотреть и повторить предыдущие команды                                                                                   |
| `clear`                                          | Очистка экрана терминала                                                                     | Для удобства чтения вывода                                                                                                            |
| `echo $SHELL`                                    | Показывает используемую оболочку (например, `/bin/bash`)                                     | Удобно проверить текущую активную shell                                                                                                |
| `chsh`                                           | Изменение shell пользователя по умолчанию                                                   | Позволяет сменить оболочку                                                                                                            |
| `alias ut=uptime`                                | Создание псевдонима `ut` для команды `uptime`                                               | Упрощает ввод длинных команд                                                                                                          |
| `ut`                                             | Выполнение команды `uptime` через псевдоним                                                | Выводит время работы системы и нагрузку                                                                                              |
| `export ROOM=101`                                | Установка переменной окружения `ROOM` с значением `101`                                    | Переменная доступна в текущей сессии и дочерних процессах                                                                             |
| `echo $ROOM`                                     | Вывод значения переменной `ROOM`                                                           | Правильный способ получить значение переменной                                                                                       |
| `echo $PATH`                                     | Показывает содержимое переменной PATH                                                      | Список директорий, где оболочка ищет исполняемые файлы                                                                                |
| `env`                                            | Вывод всех переменных окружения с их значениями                                           | Просмотр и диагностика окружения                                                                                                      |
| `which terraform`                                | Показывает путь к исполняемому файлу команды `terraform` (если есть)                       | Используется для поиска расположения программы                                                                                        |
| `export PATH=$PATH:/opt/terraform`                | Добавляет `/opt/terraform` в переменную PATH                                              | Делает исполняемые файлы в этой папке доступными в терминале                                                                          |
| `ls -l /sbin/init`                               | Информация о файле `/sbin/init` (обычно символическая ссылка на systemd)                     | Отражает, что инициализация осуществляется через systemd                                                                              |
| `sudo -l /sbin/init`                             | Проверка возможности запуска `/sbin/init` с sudo                                          | Выясняет права пользователя на выполнение этой команды с повышенными привилегиями                                                      |
| `sudo systemctl get-default`                      | Показ текущего целевого уровня загрузки systemd (например, `graphical.target`)             | Определяет, какой режим загрузки установлен по умолчанию                                                                             |
| `sudo systemctl set-default multi-user.target`   | Установка по умолчанию текстового режима без GUI                                           | При следующей перезагрузке загрузится многопользовательский режим без графики                                                        |
| `sudo systemctl isolate multi-user.target`        | Немедленное переключение в текстовый многопользовательский режим                          | Переключается на runlevel 3 без перезагрузки                                                                                         |
| `sudo systemctl isolate graphical.target`         | Немедленное переключение в графический режим                                             | Возвращение к runlevel 5 без перезагрузки                                                                                            |
| `telinit 3`                                      | Переключение runlevel на 3 (текстовый режим), устаревшая команда                         | Старый способ переключения режимов (для init), в systemd предпочтительнее systemctl isolate                                            |
| `telinit 5`                                      | Переключение runlevel на 5 (графический режим)                                           | Аналогично предыдущему                                                                                                                |
| `man uptime`                                     | Открывает руководство по команде `uptime`                                                | Подробная документация с описанием и примерами                                                                                       |
| `uptime --help`                                 | Краткая справка по команде `uptime`                                                      | Быстрая помощь по доступным опциям                                                                                                  |
| `whatis uptime`                                 | Однострочное описание команды `uptime`                                                   | Быстрый способ узнать назначение команды                                                                                            |
| `apropos uptime`                                | Поиск по ключевому слову `uptime` среди страниц руководства                             | Полезно для поиска всех связанных команд и справок                                                                                  |
